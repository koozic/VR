# 검증 체크리스트

이 문서는 변경 후 무엇을 확인해야 하는지 정리하는 검증판이다.  
프로젝트별 명령은 확인된 뒤에만 기록하고, 불확실하면 `작성 필요`로 둔다.

## 공통 검증 원칙

| 원칙 | 설명 |
| --- | --- |
| 변경 범위 기반 | 수정한 영역에 맞는 최소 검증을 선택한다. |
| 자동 검증 우선 | 테스트와 빌드가 있으면 먼저 실행한다. |
| 수동 검증 기록 | 화면, API, 로그 확인은 어떤 조건에서 봤는지 기록한다. |
| 실패도 자산 | 실패한 검증은 지우지 말고 원인과 다음 행동을 적는다. |

## 작업 종료 전 체크리스트

- [ ] 수정한 파일 목록을 확인했다.
- [ ] 변경 영역에 맞는 자동 검증을 실행했다.
- [ ] 자동 검증이 없으면 수동 확인 방법을 기록했다.
- [ ] 실패한 검증이 있으면 오류 메시지와 원인을 기록했다.
- [ ] 환경 문제로 검증하지 못한 항목은 이유를 기록했다.
