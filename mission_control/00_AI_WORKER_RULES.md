# AI 작업자 행동 수칙

이 문서는 AI 작업자가 프로젝트에서 작업할 때 지켜야 할 운영 규칙이다.  
목표는 빠른 작업이 아니라 컨텍스트 유지, 범위 이탈 방지, 이력 추적이다.

## 작업 중 행동 규칙

| 규칙 | 설명 |
| --- | --- |
| 범위 고정 | 요청과 직접 관련 없는 파일은 수정하지 않는다. |
| 먼저 관찰 | 변경 전 관련 파일, 테스트, 설정을 읽는다. |
| 작게 변경 | 한 번에 넓은 구조 변경을 하지 않는다. |
| 사용자 변경 보호 | 내가 만들지 않은 변경을 임의로 되돌리지 않는다. |
| 근거 기반 판단 | 모르는 내용은 추측하지 않고 확인하거나 `작성 필요`로 둔다. |
| 실패 기록 | 실패한 명령, 실패 이유, 우회 방법을 기록한다. |
| 보안 우선 | 비밀번호, 토큰, 개인 정보, 운영 URL을 문서에 평문으로 남기지 않는다. |

## 금지 사항

- [ ] 요청받지 않은 대규모 리팩터링을 하지 않는다.
- [ ] 관련 없는 파일 포맷팅을 하지 않는다.
- [ ] 기존 사용자 작업을 임의로 삭제하거나 되돌리지 않는다.
- [ ] 비밀키, 계정 정보, 내부 운영 정보를 문서에 평문으로 적지 않는다.
- [ ] 검증하지 않은 내용을 완료된 사실처럼 보고하지 않는다.
- [ ] "나중에 하면 됨" 같은 모호한 표현만 남기고 인수인계를 끝내지 않는다.

## 보고 형식

작업 종료 보고에는 최소한 아래 항목을 포함한다.

| 항목 | 내용 |
| --- | --- |
| 변경 요약 | 무엇을 바꿨는지 1-3문장 |
| 변경 파일 | 주요 파일 목록 |
| 검증 | 실행한 테스트, 빌드, 수동 확인 |
| 남은 위험 | 아직 확인하지 못한 부분 |
| 다음 작업 | 바로 이어서 할 수 있는 구체적 작업 |
| 커밋 메시지 | 한 줄 커밋 메시지 제안 |
