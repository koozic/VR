# 의사결정 기록

이 문서는 아키텍처와 운영상 중요한 결정을 ADR 형식으로 기록한다.  
되돌리기 어렵거나 여러 영역에 영향을 주는 선택은 반드시 기록한다.

## ADR 작성 기준

아래 중 하나라도 해당하면 새 ADR을 작성한다.

- [ ] 프레임워크, 라이브러리, 외부 서비스를 새로 도입했다.
- [ ] 데이터 저장 방식이나 API 계약이 바뀌었다.
- [ ] 인증, 권한, 보안 정책이 바뀌었다.
- [ ] 배포, 실행, 운영 방식이 바뀌었다.
- [ ] 기존 구조를 크게 이동하거나 책임 경계를 바꿨다.
- [ ] 성능, 비용, 안정성 사이의 트레이드오프를 결정했다.
